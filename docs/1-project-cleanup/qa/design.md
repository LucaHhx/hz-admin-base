# QA 技术方案 -- 项目清理 (project-cleanup)

> 需求: 1-project-cleanup | 角色: qa

## 概述

本需求为纯清理操作，不涉及新功能开发。QA 重点验证清理后项目的完整性和可用性。

## 测试策略

### 测试类型

| 类型 | 说明 | 优先级 |
|------|------|--------|
| 编译验证 | 前后端均可正常编译 | P0 |
| 启动验证 | 后端服务可正常启动，前端 dev server 可运行 | P0 |
| 功能回归 | 核心系统功能（登录、角色、菜单、API 管理）不受影响 | P0 |
| 标识一致性 | 所有项目名称统一为 HAB | P1 |
| 死代码检查 | 无残留的 example/plugin 引用 | P1 |

### 测试范围

**需要验证的:**
- 后端 `go build` 编译通过
- 后端服务启动无 panic
- 前端 `npm run build` 编译通过
- 用户登录/登出正常
- 菜单树加载正常（无 example 相关菜单）
- 初始化数据库（首次 init）流程正常
- config.example.yaml 中的配置项名称已更新

**不需要验证的:**
- 系统核心模块的详细功能测试（不在本次清理范围内）
- 性能测试
- 安全测试

## 关键决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 测试方式 | 手动验证 | 清理操作一次性执行，不需要自动化测试 |
| 验证环境 | 本地开发环境 | 清理验证不需要独立测试环境 |
